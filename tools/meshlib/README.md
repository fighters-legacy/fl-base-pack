# fl_meshlib

Shared procedural-aircraft mesh helpers, extracted from the F-5E build script so the F-16A, T-38A
and every aircraft after them start from common code instead of a copy-paste. MIT-licensed (more
permissive than the pack's CC-BY-4.0 content, so other packs and the sample template can reuse it).

## Layout

    src/fl_meshlib/
      airfoil.py    NACA symmetric section          (pure math)
      curves.py     smoothstep, lerp                 (pure math)
      stations.py   station-table interpolation      (pure math)
      check.py      cheap local .glb sanity checks   (pure Python; no Blender)
      loft.py       trapezoidal panel, ring bridge   (Blender: bmesh)
      scene.py      clear/finish/material/hull/empty  (Blender: bpy)
      uvatlas.py    planar UVs (atlas islands later)  (Blender: bpy)
      damage.py     deterministic _b damage variant   (Blender: bpy)
      export.py     GLB export, texture URI patch, decimate (Blender + pure)

The pure-math modules import with no third-party dependencies and are unit-tested without Blender;
the Blender-facing modules import `bpy`/`bmesh`/`mathutils` and only load inside Blender. `__init__`
imports nothing, so `from fl_meshlib import airfoil` never drags in `bpy`.

## Use from a build script

Build scripts add the library to Blender's Python path from their own location, so there is no
install step under `blender --background`:

    _REPO_ROOT = Path(__file__).resolve().parents[2]
    sys.path.insert(0, str(_REPO_ROOT / "tools" / "meshlib" / "src"))
    from fl_meshlib import export, loft, scene, uvatlas

External consumers can instead:

    pip install "fl-meshlib @ git+https://github.com/fighters-legacy/fl-base-pack@<tag>#subdirectory=tools/meshlib"

## Tests

    cd tools/meshlib
    PYTHONPATH=src python -m pytest tests/ -q          # pure math + GLB checks, no Blender
    pip install bpy && PYTHONPATH=src python -m pytest tests/test_determinism.py

`test_determinism.py` builds the F-5E twice and asserts the outputs are byte-identical — the
regression check behind the pack's "generated" provenance mode. It skips where `bpy` is absent.
Set `FL_MESHLIB_MATCH_COMMITTED=1` (with the pinned Blender the pack ships with) to also assert the
build matches the committed artifacts byte-for-byte.

## Determinism is a contract

Aircraft are regenerated byte-for-byte as a regression check. Keep it that way: seeds are explicit
parameters, never the wall clock; iterate containers in a fixed order; keep export options fixed.
